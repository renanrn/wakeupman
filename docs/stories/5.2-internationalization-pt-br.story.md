# Story 5.2: Internationalization (i18n) - pt-BR Support

## Problem Statement
The application was previously hardcoded with English strings, limiting its usability for non-English speakers, particularly in the Brazilian market where safety tools like this are highly relevant.

## Acceptance Criteria
- [x] Extract all hardcoded strings from Compose screens and Services to `res/values/strings.xml`.
- [x] Implement a complete translation for Brazilian Portuguese in `res/values-pt-rBR/strings.xml`.
- [x] Refactor all UI components to use `stringResource(R.string.key)`.
- [x] Refactor `VigilanceService` to use localized notification text via `getString(R.string.key)`.
- [x] Ensure the app correctly switches language based on device system settings.

## Tasks
- [x] Create `strings.xml` with English base strings.
- [x] Create `values-pt-rBR` directory and translated `strings.xml`.
- [x] Refactor `DashboardScreen.kt`, `PermissionOnboardingScreen.kt`, `BatteryOptimizationGuideScreen.kt`, `AlertActivity.kt`, and `HistoryScreen.kt`.
- [x] Refactor `VigilanceService.kt`.
